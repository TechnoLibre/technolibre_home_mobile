# App i18n

The app uses a lightweight hand-rolled i18n system in `src/i18n/`. There is no external
library — the entire runtime is under 50 lines of TypeScript.

## Architecture overview

```
src/i18n/
├── index.ts   — runtime: getCurrentLocale(), setLocale(), t()
├── fr.ts      — French translations (source of truth / fallback)
└── en.ts      — English translations
```

Each locale file exports a single `translations` record with flat dot-separated keys:

```typescript
export const translations: Record<string, string> = {
    "nav.home": "Accueil",
    "label.font_size": "Taille de police — {size}",
    // …
};
```

`index.ts` imports both records into a `DICTS` map keyed by `Locale` (`"fr" | "en"`).

## How `t()` works

```typescript
t(key: string, params?: Record<string, string | number>): string
```

1. Read the current locale from `localStorage` under the key `"app_lang"` (default `"fr"`).
2. Look up `key` in that locale's dictionary.
3. If missing: fall back to the French dictionary, then to the raw key string itself, and
   emit a `console.warn` so missing keys are visible during development.
4. Substitute named placeholders: every occurrence of `{paramName}` in the resulting string
   is replaced with the corresponding value from `params`.

Example with interpolation:

```typescript
t("label.font_size", { size: 16 })
// fr → "Taille de police — 16"
// en → "Font size — 16"

t("error.note_key_not_found", { label: "note" })
// fr → "Échec de la récupération des notes du stockage local."
// en → "Failed to retrieve notes from local storage."
```

Placeholders are replaced with `String.replaceAll`, so a placeholder appearing multiple
times in the same value is replaced consistently. `t()` never throws and never returns
`undefined`.

## How to add a new translation key

1. Add the key to `src/i18n/fr.ts` in the appropriate comment section:

   ```typescript
   // My section
   "my_section.my_key": "Texte en français",
   ```

2. Add the same key to `src/i18n/en.ts`:

   ```typescript
   // My section
   "my_section.my_key": "English text",
   ```

3. Use the key in a component or service:

   ```typescript
   import { t } from "../../i18n";

   const label = t("my_section.my_key");
   const interpolated = t("my_section.my_key_with_param", { count: 3 });
   ```

If the key is missing from the active locale's file, `t()` falls back to the French value
and logs a warning — it never renders `undefined` or throws.

## How to add a new language

1. Create `src/i18n/xx.ts` (where `xx` is the BCP-47 tag, e.g. `"es"`) with the full
   translations record, using `fr.ts` as the template.

2. Import it in `src/i18n/index.ts` and extend the `Locale` type and `DICTS` map:

   ```typescript
   import { translations as es } from "./es";

   export type Locale = "fr" | "en" | "es";

   const DICTS: Record<Locale, Record<string, string>> = { fr, en, es };
   ```

3. Update `getCurrentLocale()` to accept the new tag:

   ```typescript
   export function getCurrentLocale(): Locale {
       const stored = localStorage.getItem(STORAGE_KEY);
       if (stored === "en" || stored === "es") return stored;
       return "fr";
   }
   ```

4. Add picker label keys to both `fr.ts` and `en.ts`:

   ```typescript
   "language.es": "Español",
   ```

5. Add the new option to the language picker component
   (`src/components/options/options_language_component/`).

## Runtime locale switching

```typescript
import { getCurrentLocale, setLocale } from "../../i18n";

// Read the active locale (synchronous — reads localStorage)
const locale: Locale = getCurrentLocale();  // "fr" | "en"

// Switch locale — persists to localStorage then reloads the page
setLocale("en");
```

`setLocale` writes to `localStorage` and immediately calls `window.location.reload()`. The
full page reload is intentional: because `t()` is called at render time (not reactively), a
reload is the simplest way to re-render all strings without introducing a global reactive
signal. The `STORAGE_KEY` is `"app_lang"`. Any stored value other than `"en"` defaults to
`"fr"`.

## Options › Language page

The locale picker lives at:

```
Options › Langue
src/components/options/options_language_component/
```

It presents two choices using the keys `language.fr` (`Français (Canada)`) and
`language.en` (`English (Canada)`). Selecting either calls `setLocale(locale)`, which
displays a brief note (`language.reload_note`) and triggers the page reload.

The route is reached from the Options main menu via the `options.language` menu entry.
